# 🚀 項目開發與優化 Log 日誌 (Project Development & Tuning Log)

## 📌 基礎資訊
- **專案名稱**：Bullet Hell DRL (彈幕地獄強化學習)
- **環境架構**：Pygame (遊戲本體) + Gymnasium (RL環境對接) + Stable-Baselines3 PPO (演算法)
- **日誌目的**：記錄優化 DRL 訓練穩定度、過濾觀測雜訊、調整難度曲線及串接獎勵機制的歷程。

## 📅 日誌內容

### 1. 遊戲核心機制微調 (Game Core Tweaks)
* **調整升級火力增長幅度**：
    * **討論背景**：原版程式碼中，玩家升級時射擊間隔（攻速）每次直接除以 `1.2`，導致中後期火力瞬間暴增。這會為環境帶來嚴重的**非平穩性 (Non-stationarity)**，Agent 會搞不懂是因為自己走位變好，還是因為數值紅利突然變強。
    * **調整更動**：保留升級回血（回復最大血量的一半）以拉長訓練戰線，但將射擊間隔加強程度從除以 `1.2` 降為 **除以 `1.05`**。同時，子彈速度增長從每次 `+2` 降為 **`+1`**，讓火力提升更加平滑，利於演算法收斂。
* **敵機左右正弦波搖擺原理解析**：
    * **機制理解**：分析了敵機 `wave_amplitude` (左右振幅) 與 `wave_offset` (隨機初始相位) 的數學原理。目前的設定下，敵機只會在原本降落位置附近進行 4~10 像素的「局部 S 型微幅扭動」以打破呆板感，而非全場大範圍移動。透過 `wave_offset` 錯開每隻敵機的擺動節奏，營造滑行視覺感。

### 2. 遊戲難度曲線放緩 (Difficulty Curve Smoothing)
* **拉長 Stage 升級時間基準**：
    * **討論背景**：原系統每經過 8 秒（`8 * FPS` 幀）就會強制提升一個 Stage 等級，導致 40 秒時畫面就塞滿跑得飛快、瘋狂射擊的敵機，後期極度混亂，Agent 容易被秒殺而學不到東西。
    * **調整更動**：在 `game.py` 的 `_tick` 中，將難度推進時間從 8 秒大幅拉長至 **20 秒 (`20 * FPS`)**。
* **優化難度參數成長公式**：
    * **調整更動**：微調 `_level_params()` 函數：
        * 將敵機與敵彈的速度每級成長係數從 `0.5` 降至 **`0.2`**。
        * 放緩生成間隔縮短趨勢（每級減 8 幀改為 **減 4 幀**），且設定最低限制為 **50 幀**（原為 40 幀）。
        * 限制同一波生成的最大數量：上限從 5 隻降至 **3 隻**，且從每 2 級加 1 隻放緩為 **每 3 級加 1 隻**。

### 3. Gym 環境與獎勵機制對接優化 (Gym & Reward Optimization)
* **串接補血包獎勵 (REWARD_HEAL)**：
    * **討論背景**：原版 `bullet_hell_env.py` 未將玩家吃到補血包的 `healed` 變數傳遞給獎勵計算器，導致 Agent 無法學會主動追逐綠色十字。
    * **調整更動**：
        1. 修改 `game.py` 的 `_tick` 與 `_update_ai`，使其同步回傳 `kill_count, hit, died, healed` 四個數值。
        2. 修改 `bullet_hell_env.py` 的 `step()`，利用 Frame Skip 累積該步內是否吃過補血包，並傳給獎勵計算器。
        3. 在 `reward.py` 中新增常數 **`REWARD_HEAL = +2.0`**。該值設定低於擊殺獎勵（`+5.0`），引導 Agent 殘血時去吃補血包，同時防止其產生「故意不殺敵、掛機等補血」的偷懶行為。
* **觀測空間特徵最佳化 (濾除星空雜訊)**：
    * **討論背景**：原版 `_capture_frame()` 捕捉背景裡隨機向下滾動的白點星星。這些星星對 CNN 卷積神經網路來說是**嚴重的干擾雜訊**，會大幅拖慢訓練速度（Agent 會花費大量步數試圖閃避沒有傷害的背景點）。
    * **調整更動**：在 `game.py` 用於 AI 畫面捕捉的 `_draw_objects()` 函數中，**移除繪製星星的迴圈**。讓人類玩家看得到的畫面保留星空，而 Agent 看到的觀測畫面保持純黑底，只保留戰機、敵機、子彈與補血包。

### 4. DRL 演算法超參數調校指南 (Hyperparameter Tuning Guide)
* **明確 PPO 參數配置位置**：
    * 確認了學習率（`learning_rate`）、採樣步數（`n_steps`）等重要超參數直接位於 `train.py` 的 `PPO(...)` 初始化宣告中。
* **參數對應效應與調校建議**：
    * **`learning_rate` (目前 `2.5e-4`)**：因影像輸入且走位容錯低，若平均獎勵震盪劇烈，建議調小至 `1e-4` 或 `5e-5` 以穩定更新。
    * **`n_steps` (目前 `2048`)**：在 `FRAME_SKIP = 4` 下相當於 8192 幀經驗。若要加強長線走位佈局，可嘗試放大至 `4096`。
    * **`batch_size` (目前 `64`)**：建議隨 `n_steps` 同步放大（如 `128` 或 `256`）以穩定每次的梯度計算。
    * **`gamma` (目前 `0.99`)**：折扣因子。因彈幕遊戲需提前規劃走位，建議保持 `0.99` 或提高至 `0.995` 以看重遠期風險。
    * **動作空間（Action Space）**：維持現有的 9 個離散動作。初期先不加入動作連續變更懲罰，避免抑制 Agent 的極限閃彈本能，視後續是否有「原地鬼畜抖動」再行微調。
