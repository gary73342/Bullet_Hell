# 2026-06-04_08 外部遊戲環境整合

## 背景

原版遊戲在設計上有潛在缺陷，為了驗證 reward 設計與訓練流程的通用性，引入外部 bullet hell 遊戲作為對照實驗環境，並讓報告有更多可寫的比較素材。

來源：`github.com/michael-pacheco/deep-learning-bullet-hell-environment`，clone 至 `game_ext/`。

## 新增內容

### game_ext/
- Clone 自外部 repo，主要使用 `game/main.py`、`game/sprites.py`、`game/levels.py`
- 修正 `game/helpers.py`：Python 2 舊語法 `except (pygame.error, message)` 改為 Python 3 相容寫法

### env_ext/\_\_init\_\_.py
- Package 初始化，export `ExtBulletHellEnv`

### env_ext/ext_env.py
- 將外部遊戲包裝為標準 Gymnasium 環境
- **動作空間**：9 個離散動作（8 方向 + 不動），與原版遊戲一致，讓 agent 與人類操作條件相同
  - 移動方式改為直接設定 `hitbox.xMove / yMove`，繞過鍵盤注入機制，支援斜向移動
- **觀測空間**：`Dict`，格式與原版 `bullet_hell_env.py` 一致
  - `pixels`：(12, 84, 84)，4 幀堆疊 RGB 截圖
  - `state`：(62,) 結構化向量，含玩家 x/y、最近 20 顆子彈座標、最近 10 隻怪物座標
- **Reward**：移植自 `env/reward.py`，依外部遊戲特性調整
  - 保留：存活獎勵、死亡懲罰、靜止懲罰、貼邊懲罰、Potential-Based 排斥場
  - 移除：補血獎勵、擊殺獎勵（外部遊戲為一血秒死，核心目標純閃彈）
- **Reset**：Soft reset（清空 sprite group + `levelInit` + `variableInit` + `initGame`），不重建 Main 物件
- **Headless 模式**：訓練時設定 `SDL_VIDEODRIVER=dummy`

### train.py
- 新增 `--env` 參數（`original` / `ext`），可切換訓練環境
- 預設維持 `original`

### evaluate.py
- 新增 `--env` 參數（`original` / `ext`）
- 新增 `--model` 參數指定模型路徑
- `info` 列印依環境調整（ext 無血量欄位）

## 外部遊戲與原版遊戲差異

| 項目 | 原版遊戲 | 外部遊戲 |
|---|---|---|
| 動作數 | 9 | 9（本次調整後） |
| 玩家血量 | 5 | 1（一血秒死） |
| 補血 | 有 | 無 |
| 升級 | 有 | 無 |
| 玩家射擊 | 無 | 有（自動） |
| 螢幕尺寸 | 原版尺寸 | 512×512 |

## 效能調整

- 外部遊戲 `MainLoop` 開頭有 `pygame.time.wait(30)`（為人類遊玩設計），訓練時強制設為 0
- `variableInit()` 每次 reset 會將 `ms_per_round` 重設回 30，在 `_soft_reset()` 中覆蓋修正
- 調整後速度從 22 steps/s 提升至 96 steps/s
