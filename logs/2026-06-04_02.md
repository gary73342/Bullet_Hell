# 2026-06-04_02 Gym 層 Bug 修復與 Reward 調整

## 修復項目

### Bug 1：`current_frame` 從未遞增
- **檔案**：`env/bullet_hell_env.py`
- **問題**：`step()` 裡 `self.current_frame` 始終為 0，導致 `reward.py` 的里程碑獎勵永遠不觸發
- **修法**：在 `step()` 計算完 reward 後加入 `self.current_frame += 1`

### Bug 2：`SDL_VIDEODRIVER = "dummy"` 污染 process 環境
- **檔案**：`game/game.py`
- **問題**：headless 模式下將 `SDL_VIDEODRIVER` 設為 `dummy` 但從未還原，導致同 process 中後續建立的 `render_mode="human"` Game 無法開視窗
- **修法**：`pygame.init()` 前先存下原始值，init 完成後立刻還原

### Bug 3：`MILESTONE_FRAMES` 單位錯誤
- **檔案**：`env/reward.py`
- **問題**：`current_frame` 計的是 env steps，`MILESTONE_FRAMES = 600` 實際對應 40 秒而非 10 秒
- **修法**：改為 `150`（150 steps × frame_skip 4 = 600 遊戲幀 = 10 秒）

### Bug 4：反抖動偵測變數命名對調
- **檔案**：`env/reward.py`
- **問題**：`is_shaking_x` 實際偵測的是上下（Y 軸），`is_shaking_y` 偵測的是左右（X 軸），名稱與軸向相反
- **修法**：對調變數名稱，使命名與 ACTION_MAP 一致

## 功能調整

### 動態被打懲罰（依玩家等級）
- **檔案**：`env/reward.py`、`env/bullet_hell_env.py`
- **設計**：前期鼓勵積極進攻，高等級後加重躲避要求
- **公式**：`hit_reward = -4 - 2 × (player_level // 5)`
- **對照表**：

| 等級  | 被打懲罰 |
|-------|---------|
| 0–4   | -4      |
| 5–9   | -6      |
| 10–14 | -8      |
| 15–19 | -10     |
| 20    | -12     |

- **實作**：`RewardCalculator.calculate()` 新增 `player_level` 參數；`BulletHellEnv.step()` 傳入 `self._game.player_level`
