## 架構說明

**雙模態輸入設計（CNN + MLP）**

- **CNN**（12 channels = 4 frame × RGB）
  - 捕捉全局視覺空間關係
  - 4 幀堆疊讓 agent 感知子彈移動方向（單幀無法判斷）
- **MLP**（69 維結構化狀態）
  - 玩家座標 (2) + HP (1)
  - 最近 20 顆子彈位置 (40)
  - 最近 10 隻敵人位置 (20)
  - 最近 3 個補血道具位置 (6)
- **融合**：CNN 特徵 (4480 維) + MLP 特徵 (64 維) → 全連接層 (512 維) → Actor / Critic

> CNN 看全局態勢，MLP 給精確位置資訊，兩者互補。

---

## Reward Function 說明

| 成分 | 數值 | 設計意圖 |
|------|------|---------|
| 存活 | +0.1/step | 鼓勵活越久越好 |
| 擊殺 | +5.0/kill | 鼓勵主動攻擊 |
| 被打 | -4.0（等級越高懲罰越重） | 鼓勵閃避 |
| 死亡 | -50.0 | 最嚴重失敗訊號 |
| 靜止 | -1.0（超過 8 步不動） | 防止角落躲避策略 |
| 貼邊 | -2.0 | 防止龜縮邊界 |
| Potential-Based Shaping | 動態（指數衰減場） | 子彈/敵人周圍提供密集閃避信號 |

Potential-Based Shaping 採用 Ng et al. (1999) 的設計，理論上不改變最優策略，只提供更密集的學習信號。

## 教授可能的問題與回答

**Q：reward 方差很大，是沒學好嗎？**
Bullet hell 遊戲每局子彈密度和位置具有隨機性，單局 reward 方差高屬正常現象。從 smoothed 曲線可以看到明確的上升趨勢。

**Q：為什麼用 Potential-Based Shaping？**
直接 reward 只在被打或死亡時給訊號，reward 過於稀疏。Potential-based shaping 提供連續的密集信號，讓 agent 更快學會遠離危險，且理論上不影響最優策略。

**Q：value loss 沒有完全收斂？**
Value loss 代表 Critic 對 return 的估計誤差。Critic 估計精度還在追趕，但 Actor 的 reward 表現已趨於穩定，說明 policy 本身已相對收斂。

**Q：為什麼訓練到第七次停止？**
train7 的 reward 曲線無明顯上升趨勢，與 train6 峰值相比無顯著差距，判定在當前設定下繼續訓練的邊際效益已低，因此以 train6 最佳 checkpoint 作為最終模型。
