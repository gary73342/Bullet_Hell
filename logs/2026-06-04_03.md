# 2026-06-04_03 獎勵函數改動：排斥場與連續生存獎勵

## 改動一：Potential-Based Repulsive Field

### 問題
原本的位置獎勵將代理引導向固定的「甜蜜點」（畫面中央 3/4 處），違反 Potential-Based Reward Shaping 的前提：彈幕遊戲的最優策略是動態迴避，而非待在固定座標。

### 理論依據
Ng et al. (1999) *Policy Invariance Under Reward Transformations* 指出，只有符合以下形式的 shaping 才能保證不改變最優策略：

```
F(s, s') = γΦ(s') − Φ(s)
```

### 修法
- **移除**：`_SWEET_X`、`_SWEET_Y`、`_MAX_DIST`、`REWARD_POS_MAX`
- **新增**：排斥場 potential function

```
Φ(s) = −K_bullet × Σ 1/(d_bullet_i + ε)
       −K_enemy  × Σ 1/(d_enemy_i  + ε)
```

子彈與敵機的係數相同（`K = 300.0`），因為兩者碰撞傷害一致（均扣 1 HP）。只計算距離 200px 以內的威脅。

- **改動檔案**：`env/reward.py`、`env/bullet_hell_env.py`

## 改動二：連續生存獎勵取代里程碑

### 問題
每 150 步才觸發一次的 `+15` 獎勵屬於稀疏 reward，GAE 折扣後遠處的功勞幾乎消失（`0.99^150 ≈ 0.22`），代理難以學習長期生存策略。

### 修法
- **移除**：`REWARD_TIME_MILESTONE = 15.0`、`MILESTONE_FRAMES = 150`、里程碑 if 判斷、`current_frame` 參數
- **新增**：`REWARD_SURVIVE = 0.1`，每步存活直接累加

總獎勵規模不變（`15 / 150 = 0.1`），但從稀疏轉為密集信號，信用分配問題消除。

- **改動檔案**：`env/reward.py`、`env/bullet_hell_env.py`
