## 論文參考

參考兩篇 DeepMind DQN 論文：
- Playing Atari with Deep Reinforcement Learning (2013)
- Human-level control through deep reinforcement learning, Nature (2015)

### 論文的 Reward 做法

- Reward 直接使用遊戲分數的變化量，不做額外設計
- Nature 版提出 **Reward Clipping**：所有 reward 壓縮至 [-1, 0, +1]，提升訓練穩定性
- 兩篇論文均使用 4 幀堆疊、Frame Skip 4（與本專題設計一致）
- 兩篇論文均使用灰階（本專題因顏色語義明確，改用 RGB）

### 與本專題的差異

論文跑 49 款遊戲，不能為單一遊戲客製化 reward。本專題只有一款自訂遊戲，可以針對性地設計 reward function，引導 AI 形成正確的遊玩習慣。

## 最終 Reward Function 設計

### 各項說明

| 項目 | 數值 | 說明 |
|------|------|------|
| 擊殺普通敵人 | +5.0 | 主要正向驅動力 |
| 被子彈擊中 | -10.0 | 懲罰被打 |
| 死亡 | -50.0 | 終局懲罰 |
| 連續靜止 > 10 步 | -0.5 | 懲罰完全不動，避免 AI 躲角落刷存活 |
| 靠近甜蜜點 | +0.02（最大值） | 鼓勵 AI 待在中間偏下，增加反應空間 |

### 移除的項目

- 每幀存活 reward（+0.01）：拿掉，避免 AI 消極求生；死亡懲罰本身已足夠驅動存活
- 擊殺 Boss reward（+20.0）：Boss 尚未實作，暫時移除

### 甜蜜點定義

```
x = SCREEN_WIDTH / 2   = 240px（水平正中央）
y = SCREEN_HEIGHT × 0.75 = 480px（垂直 3/4 處，偏下方）
```

位置 reward 公式：
```
reward = 0.02 × (1 - distance_to_sweet_spot / max_distance)
```

越靠近甜蜜點越高，待在邊角幾乎為 0。

### Reward Clipping

目前未實作，但論文建議的 Reward Clipping（壓到 ±1）在訓練不穩定時可考慮加入。

### 數值比例說明

- 輔助 reward（靜止懲罰 + 位置獎勵）的每秒影響遠小於主要 reward，不會蓋過擊殺和被打的訊號
- 目前數值為初始設定，預計訓練後依實際 reward 曲線調整

## 實作

- `env/__init__.py`：套件入口，export `RewardCalculator`
- `env/reward.py`：`RewardCalculator` 類別，提供 `reset()` 和 `calculate()` 方法

### calculate() 介面

```python
reward = calc.calculate(
    player_x = int,   # 玩家目前 x 座標
    player_y = int,   # 玩家目前 y 座標
    kills    = int,   # 這個 step 內擊殺數
    hit      = bool,  # 這個 step 內是否被擊中
    died     = bool,  # 這個 step 內是否死亡
)
```
